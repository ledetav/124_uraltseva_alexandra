Пользуясь технологией разработки через тестирование (TDD), реализуйте систему учета платежей по договорам, удовлетворяющую следующим требованиям 

--- FEATURES.
— добавление договора с заданным номером (строка) и датой заключения (строка формата "YYYYMMDD", где Y - год, M - месяц, день). +
— возможность регистрировать платежные документы типа "Платежное поручение" и "Банковский ордер". +
— добавление платежного докуменда с указанной суммой, номером, типомдокумента, номером договора и датой платежа. +
— Сумма задается целым положительным числом копеек; +
— номер платежного документа - положительное целое число; +
— дата платежа - строка формата "YYYYMMDD"; +
— тип документа задается перечислением. +
— поиск всех платежей по договору с заданным номером. +
— вычисление суммы всех платежей по договору с заданным номером. +
— уделение платежей с заданным номером, номером договора и датой. + 
— получение списка всех договоров с их суммарными суммами платежей.
— получение списка всех платежей.

--- ERRORS.
— попытка добавить договор с уже существующим номером - ошибка. +
— попытка добавить договор с пустым номером - ошибка. + 
— попытка удалить несуществующий платеж - ошибка.
— попытка добавить платеж с уже существующим номером договора, номером документа, типом документа и датой - ошибка.
— null в качестве аргументов - ошибка.
— отрицательная или нулевая сумма платежа - ошибка.
— обработка ошибок осуществляется с помощью выбрасывания исключений с соответствующим сообщением.
